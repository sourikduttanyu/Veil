# privacap Dashboard

## Product Purpose
A monitoring dashboard for a privacy-preserving ad frequency capping system. Shows how differential privacy works in practice: ads get capped without tracking individual users. Primary audience: engineers demonstrating the system, recruiters evaluating technical depth, and curious non-technical visitors.

## Users
- **Recruiters / hiring managers**: non-technical, need plain English explanations of every number. Want to understand what the system does, not how it works mathematically.
- **Engineers**: building or evaluating the system, understand DP math, want data density.
- **First-time visitors**: loaded by a portfolio link, have zero context, need onboarding via the UI itself.

## Register
product

## Brand tone
Clear, confident, educational. No jargon without immediate plain-English translation. Privacy-forward: every label should reinforce that user identity is never stored.

## Anti-references
- Generic SaaS dashboards with hero metrics
- Crypto/web3 dark neon aesthetics
- Academic paper aesthetic

## Core value proposition
"Frequency caps without surveillance" — the system knows an ad group has been seen too many times without knowing who saw it.
